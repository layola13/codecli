# 代码库索引计划

## 当前实现状态

已落地为 Claude Code 内置命令 `/index`，而不是独立的外部 Python 工具。

当前实现位置：

- `src/commands/index/`
- `src/indexing/`

当前运行特征：

- 索引器实现语言为 TypeScript，直接内置到 Claude Code 命令体系
- 输出仍然是语言无关的 Python skeleton 工程，便于 LLM 阅读
- 不依赖 RAG、向量库、Neo4j、tree-sitter、外部 Python 包管理
- 当前仓库已可直接生成 `.code_index/`
- 同时生成面向 Claude / Codex 的轻量 skill 文档，统一引用这份共享索引

当前命令入口：

```text
/index
/index src
/index . --output .code_index
/index --max-file-bytes 1048576
```

## 目标

构建一个内置在 Claude Code 中的代码索引器，扫描任意代码库并生成两类产物：

1. 语言无关的逻辑骨架 Python 工程
2. 机器可读的结构化索引文件

该方案不依赖 RAG、向量库或 Neo4j。索引本身就是可读的工程与数据。

## 核心原则

- 输出优先服务 LLM 阅读，而不是服务传统搜索系统
- 骨架保留模块结构、类/函数层次、调用链、依赖关系，去掉实现细节
- 索引和骨架要能从同一套中间表示生成，避免双份逻辑
- v1 先把当前仓库最主要的 TS/JS 路径做扎实，再保留多语言扩展位
- 所有语言最终收敛到统一 IR，而不是直接在 emitter 中写语言分支

## v1 范围

- 输入语言
  - TypeScript
  - TSX
  - JavaScript
  - JSX
  - Python
- 输出内容
  - `skeleton/`：镜像原目录结构的 `.py` 骨架文件
  - `index/manifest.json`：全局摘要
  - `index/modules.jsonl`：模块级索引
  - `index/symbols.jsonl`：符号级索引
  - `index/edges.jsonl`：调用/导入/继承边
  - `index/summary.md`：给人和模型快速预览
- v1 不做
  - 类型精确求值
  - 跨文件精确静态绑定
  - 增量索引缓存
  - 图数据库写入
  - 嵌入向量检索

## 输出目录设计

默认输出目录：`.code_index/`

目录结构：

```text
.code_index/

.claude/

        └── SKILL.md

.codex/
└── skills/
    └── code-index/
        └── SKILL.md
```

## 统一 IR

索引器内部统一使用以下抽象：

- `ModuleIR`
  - 源路径
  - 相对路径
  - 语言
  - imports
  - exports
  - classes
  - functions
- `ClassIR`
  - 名称
  - bases
  - depends_on
  - methods
  - source_lines
- `FunctionIR`
  - 名称
  - qualified_name
  - 参数
  - 返回值
  - decorators
  - calls
  - awaits
  - raises
  - 是否 async
  - 是否 public
  - source_lines
- `EdgeIR`
  - kind: `imports` / `calls` / `inherits` / `depends_on`
  - source
  - target
  - source_file
  - source_symbol
  - metadata

## 解析策略

### TypeScript / JavaScript

当前实现为启发式解析：

- 提取 import/export
- 提取 class / method / function / arrow function
- 提取 constructor 参数作为依赖候选
- 提取 call expression / new expression / await expression
- 提取 extends / implements

### Python

当前实现为缩进 + 头部解析：

- 提取 imports
- 提取 class / function / async function
- 提取 decorator / return annotation
- 提取调用表达式

### Generic Fallback

对未支持或解析失败文件：

- 保留基础文件元数据
- 输出一个最小骨架模块
- 在索引中记录 `parse_mode=fallback`

## Skeleton 生成规则

- 每个源文件生成一个对应 `.py` 文件
- 目录结构镜像原仓库
- 文件头保留：
  - source path
  - language
  - exports
  - imports summary
- 类中保留：
  - bases
  - depends_on
  - methods
- 函数中保留：
  - 参数
  - 返回值
  - calls
  - awaits
  - raises
  - source_line
- 方法体统一使用 `...`

## JSON Index 设计

### modules.jsonl

每行一个模块：

- module_id
- path
- lang
- imports_count
- classes_count
- functions_count
- parse_mode

### symbols.jsonl

每行一个符号：

- symbol_id
- module_id
- kind: `class` / `function` / `method`
- qualified_name
- signature
- source_lines

### edges.jsonl

每行一条边：

- edge_id
- kind
- source_symbol
- target
- source_file
- line_start
- line_end

## 实施顺序

### Phase 1

- 内置命令脚手架
- IR 数据结构
- 文件发现与忽略规则
- emitter
- TS/JS parser
- Python parser
- generic fallback

### Phase 2

- 输出 manifest / jsonl / summary
- 生成当前仓库索引样例
- 修正 TS/JS 边界情况

### Phase 3

- 增量更新
- 更多语言 parser
- 符号解析增强

## 验收标准

- 可在当前仓库运行并生成 `.code_index/`
- `src/` 下 TS/JS 文件能稳定生成对应 Python skeleton
- skeleton 中能看出模块、类、函数、调用链
- JSON 索引可枚举模块、符号、边
- 任一解析失败文件不会中断全局构建
