# 源码直编改动记录

本文记录当前 `2.1.88+local.3` 相对原始发布包，为了支持“直接从 `src/` 用 Bun 打包”而做的主要改动。

## 1. 目标变化

原先主要依赖：

- 发布包里的 `cli.js`
- `scripts/patch-cli-sidecar.mjs`
- sidecar 方式把新增命令注入到发布入口

现在默认本地构建改为：

- 入口：`src/entrypoints/cli.tsx`
- JS bundle：`dist/claudecode.js`
- 二进制：`dist/claudecode`

也就是说，默认 `build:bun` / `compile:bun` 都输出到 `dist/`，不是项目根目录。

本次主要参考：

参考后补齐了源码直编所需的缺失文件、依赖和兼容层，但保留了本仓库额外新增的功能：

- `/index`
- `/pin`

## 3. 构建链调整

### `package.json`

主要调整：

- 版本改为 `2.1.88+local.3`
- 增加源码直编依赖
- 增加 `postinstall`
- 默认脚本切换为源码直编
- 保留发布包 patch 编译作为兼容兜底

当前关键脚本：

```json
{
  "build:bun": "bun scripts/bun-build.mjs --source",
  "compile:bun": "bun scripts/bun-build.mjs --compile --source",
  "build:bun:published": "bun run build:command-bundle && bun run patch:cli-sidecar && bun scripts/bun-build.mjs --published",
  "compile:bun:published": "bun run build:command-bundle && bun run patch:cli-sidecar && bun scripts/bun-build.mjs --compile --published"
}
```

### `scripts/bun-build.mjs`

新增了两种入口模式：

- `--source`：走 `src/entrypoints/cli.tsx`
- `--published`：走发布包 `cli.js`

这样本地开发默认不再强依赖 `cli.js patch`。

### `scripts/postinstall.sh`

新增 `postinstall` 用于补齐源码直编时缺的本地环境，包括：

- `@ant/*` stub
- `@anthropic-ai/sandbox-runtime` 本地入口修正

## 9. 当前推荐用法

安装依赖：

```bash
AUTHORIZED=1 bun install
```

构建 JS bundle：

```bash
bun run build:bun
```

构建本地二进制：

```bash
bun run compile:bun
```

运行新版本：

```bash
./dist/claudecode
```

或者：

```bash
bun dist/claudecode.js
```

注意：

- 默认产物都在 `dist/`
- 当前项目根目录没有默认生成 `./claudecode`

## 11. 结论

当前仓库已经从“必须依赖发布包 `cli.js` patch 才能本地可用”，切换为“默认直接从 `src/` 做 Bun 源码直编”。

发布包 patch 方案仍保留，但现在只是兼容兜底，不再是默认开发路径。
