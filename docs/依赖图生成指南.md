# Python 代码依赖图生成指南

本文档记录如何从 TypeScript 转译的 Python 代码生成 DOT 格式的依赖关系图。

## 问题背景

skeleton/src 目录下的 `.py` 文件是由 TypeScript 代码转译而成的，其中包含一些非标准 Python 语法（如 TypeScript 类型注解），导致标准 Python 静态分析工具（如 pyan3）无法直接处理。

## 解决方案

由于 pyan3 等工具无法处理转译后的语法，我们采用基于正则表达式的简单导入解析方法来生成依赖图。

## 使用的工具

- **pyan3**: 原计划使用的工具，但因语法问题无法工作
- **pydeps**: 同样因语法问题失败
- **自定义脚本**: 使用正则表达式解析 import 语句

## 生成步骤

### 方法一：使用自定义脚本（推荐）

```bash
# 1. 进入项目目录
cd /home/vscode/projects/claudecode/package/claude-code-2.1.88

# 2. 运行生成脚本
python3 generate_dot.py > skeleton_dependencies.dot
```

`generate_dot.py` 脚本会：
1. 扫描 `skeleton/src/` 下的所有 `.py` 文件
2. 使用正则表达式提取 `import` 和 `from ... import` 语句
3. 生成按子目录分组的 DOT 文件

### 方法二：尝试 pyan3（需要先修复语法）

如果需要更精确的调用关系分析，可以尝试使用 pyan3，但需要先修复语法错误：

```bash
# 1. 安装 pyan3
pip install pyan3

# 2. 修复语法问题（参考 structuredIO.py 中的修复）
# 需要将 TypeScript 类型注解转为 Python 类型注解

# 3. 运行 pyan3
pyan3 src/ --uses --no-defines --colored --grouped --dot > callgraph.dot
```

常见语法问题：
- `(response SDKControlResponse)` → `Callable[[SDKControlResponse], None]`
- `SDKControlRequest[request]` → `SDKControlRequest`

## 查看生成的图

### 本地查看（需要 Graphviz）

```bash
# 安装 Graphviz
# macOS: brew install graphviz
# Ubuntu: sudo apt install graphviz

# 渲染为 PNG
dot -Tpng skeleton_dependencies.dot -o skeleton.png

# 渲染为 SVG（推荐，大项目更清晰）
dot -Tsvg skeleton_dependencies.dot -o skeleton.svg
```

### DOT 文件结构

```dot
digraph skeleton {
  rankdir=LR;
  node [shape=box];

  // 按子目录分组
  subgraph cluster_root { ... }
  subgraph cluster_cli { ... }
  subgraph cluster_utils { ... }

  // 依赖边
  "moduleA" -> "moduleB";
}
```

## 注意事项

1. **导入解析精度**：当前脚本仅解析顶层导入，无法捕获条件导入或动态导入
2. **语法问题**：部分文件仍有 TypeScript 语法遗留，建议逐步清理
3. **性能**：大型项目可考虑使用 `--no-defines` 参数简化输出
