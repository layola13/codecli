# 消息时间戳和输入统计功能

## 1. 消息时间戳 (add-message-timestamps.md)

### 功能
- 在每条消息前显示 `[HH:MM]` 格式的时间戳（24小时制）
- 同一轮对话中的多个消息（如 attachment、system 等）只显示一次时间戳
- 可通过 `showMessageTimestamps` 设置开关（默认开启）

### 修改文件

#### `src/components/MessageRow.tsx`
- 添加 `showMessageTimestamps` 和 `showTimestampUuids` 两个 prop
- 在消息渲染前生成内联时间戳元素 `[HH:MM]`
- 移除了 metadata 行中的 `<MessageTimestamp />` 组件，避免重复
- 只有 `user` 和 `assistant` 类型的消息才显示时间戳

#### `src/components/Messages.tsx`
- 计算 `showTimestampUuids`：按 `user → assistant` 类型交替标记，只在类型切换时显示时间戳
- 这样同一轮用户消息被拆成多个 renderable message（system、attachment 等）时，只在第一条显示时间戳
- 将 `showTimestampUuids` 传递给每个 `MessageRow`

#### `src/components/messages/UserPromptMessage.tsx`
- 移除了传递给 `HighlightedThinkingText` 的 `timestamp` prop，避免 briefLayout 模式下重复渲染

#### `src/utils/settings/types.ts`
- 已有 `showMessageTimestamps` 设置定义（boolean，可选）

## 2. 输入统计显示 (input-stats-display.md)

### 功能
- 在聊天输入框底部显示会话统计信息
- 格式：`N msgs · usage/total P% · M reqs`
- 示例：`5 msgs · 46.5k/200k 23% · 2 reqs`
- 上下文总量：带 1M 的模型为 1,000,000，其他模型为 200,000

### 修改文件

#### `src/utils/tokens.ts`
- 新增 `calculateSessionStats(messages)` 函数
- 返回 `{ messageCount, totalTokens, totalRequests }`
- `totalTokens` 累加所有 API 响应的 token 使用量
- `totalRequests` 统计 API 请求次数

#### `src/utils/settings/types.ts`
- 新增 `showInputStats` 设置（boolean，可选，默认 true）

#### `src/components/PromptInput/Notifications.tsx`
- 导入 `calculateSessionStats` 和 `useSettings`
- 计算 `sessionStats` 并传递给 `NotificationContent`
- 在输入框底部显示统计信息，格式为 `msgs · usage/total % · reqs`
- 自动识别模型上下文大小（1M 模型 vs 200K 模型）
